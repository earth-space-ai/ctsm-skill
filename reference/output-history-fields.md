# History Output and Fields

CTSM writes output through a "history tape" abstraction. There are up to ten
tapes (`h0` through `h9`), each independently configurable for which fields
to write, how often, how to average, and whether to remap to the gridcell or
emit raw subgrid vectors. As of `ctsm5.4`, every tape splits into two files,
one for non-instantaneous fields (averages, sums, min/max) and one for
instantaneous fields.

Authoritative reference: `bld/namelist_files/namelist_definition_ctsm.xml`,
search for `category="history"`.

---

## File naming since `ctsm5.4`

Each tape number `X` produces two output files per write window:

| File | Contents |
|------|----------|
| `<case>.clm2.hXa.YYYY-MM[-DD-SSSSS].nc` | non-instantaneous fields (averaged, summed, min, max). Includes `time_bounds`. |
| `<case>.clm2.hXi.YYYY-MM[-DD-SSSSS].nc` | instantaneous fields. No `time_bounds`. Timestamp is the END of the timestep, so an instantaneous variable saved at the end of year 2023 has timestamp `2024-01-01 00:00:00`. |

Likewise for history-restart files: `rh0` becomes `rh0a` and `rh0i`.

If a tape has no instantaneous fields, the `hXi` file is not generated by
CTSM (MOSART and RTM still write it, with a placeholder field, to avoid an
"empty file" error).

This change was driven by the need to keep `time_bounds` correct for
post-processors like xarray; details in
https://github.com/ESCOMP/CTSM/pull/2445.

## Field selection: `hist_fincl1..10`, `hist_fexcl1..10`

`hist_fincl1` controls tape 0. By default, tape 0 has a long preset list of
fields suitable for monthly climate diagnostics. Add to it (in addition to
the defaults) by listing fields:

```fortran
hist_fincl1 = 'BTRAN', 'EFLX_LH_TOT_R', 'GPP'
```

To explicitly **replace** the default list with only your fields, set
`hist_empty_htapes = .true.` first:

```fortran
hist_empty_htapes = .true.
hist_fincl1       = 'TLAI', 'TSAI', 'H2OSOI'
```

To remove a field from the default tape, use `hist_fexcl1`:

```fortran
hist_fexcl1 = 'PCT_LANDUNIT'
```

The same pattern applies to `hist_fincl2..10` for tapes 1 through 9.

A field name suffix selects the averaging type for that tape:

| Suffix | Meaning |
|--------|---------|
| (none) | tape default (usually average) |
| `:I` | instantaneous (forces the field to the `hXi` file) |
| `:A` | average over the tape window |
| `:M` | minimum |
| `:X` | maximum |
| `:SUM` | sum |

Example, to put GPP as both an instantaneous snapshot on `h1i` and a daily
mean on `h1a`:

```fortran
hist_fincl2 = 'GPP', 'GPP:I'
```

## Frequency: `hist_nhtfrq`

Array, one entry per tape (1-indexed in CTSM). Negative numbers are hours,
positive are timesteps:

| Value | Meaning |
|-------|---------|
| `0` | monthly (default for tape 0) |
| `-24` | daily |
| `-1` | hourly |
| `1` | every timestep |
| `48` | every 48 timesteps |

```fortran
hist_nhtfrq = 0, -24, -1     ! tape 0 monthly, tape 1 daily, tape 2 hourly
```

---

## Output file rollover: `hist_mfilt`

How many time samples per file:

```fortran
hist_mfilt = 1, 365, 8760    ! tape 0: one month per file (since nhtfrq=0)
                              ! tape 1: 365 daily samples = one file per year
                              ! tape 2: 8760 hourly samples = one file per year
```

For monthly files, set `hist_mfilt = 1` so that each calendar month becomes
its own file.

---

## Average type: `hist_avgflag_pertape`

Tape-level default averaging flag, applied to fields with no per-field
suffix:

```fortran
hist_avgflag_pertape = 'A', 'A', 'I'    ! avg, avg, instantaneous
```

Valid values: `A` (average), `I` (instantaneous), `M` (min), `X` (max),
`SUM`.

---

## Gridded vs vector output: `hist_dov2xy`

When `.true.` (the default for every tape), CTSM remaps subgrid (patch,
column, landunit) values to the gridcell using area weights, producing
2D `lat x lon` (or 1D for unstructured grids) fields.

When `.false.` for a tape, fields keep their native subgrid dimension and
appear as 1D vectors with auxiliary `pfts1d_*`, `cols1d_*`, `land1d_*` index
variables. This is essential for PFT-level or column-level analysis (urban
canyon temperatures by wall, soil temperature by snow column, etc.).

```fortran
hist_dov2xy = .true., .false.    ! tape 0 gridded, tape 1 raw vector
```

---

## Single-point and tower cases (NEON, PLUMBER2)

Single-point CTSM cases (`I1Pt*` compsets) produce a 1D output where the
gridcell dimension is length 1. For NEON and PLUMBER2 sites, prebuilt
configurations live under `cime_config/usermods_dirs/clm/NEON/` and
`tools/site_and_regional/plumber2_usermods/`. Apply with
`--user-mods-dir clm/NEON/<SITE>` at `create_newcase` time.

NEON sites come with provided initial conditions; PLUMBER2 sites do not, so
plan a spinup before reporting any flux.

## Common namelist patterns

### Add a daily soil-moisture tape with vector output

```fortran
hist_fincl2  = 'H2OSOI', 'TSOI', 'SOILLIQ', 'SOILICE'
hist_nhtfrq  = 0, -24
hist_mfilt   = 1, 365
hist_dov2xy  = .true., .false.
```

### Custom monthly tape with only the fields you want

```fortran
hist_empty_htapes = .true.
hist_fincl1       = 'GPP', 'NEE', 'NEP', 'TLAI', 'TWS', 'QRUNOFF'
hist_nhtfrq       = 0
hist_mfilt        = 1
```

### Hourly instantaneous tape for ABL-type studies

```fortran
hist_fincl3       = 'TBOT', 'QBOT', 'TSA', 'EFLX_LH_TOT', 'FSH'
hist_nhtfrq       = 0, 0, -1
hist_mfilt        = 1, 1, 24
hist_avgflag_pertape = 'A', 'A', 'I'
```

## Discovering field names

Three sources:

1. **Code search**: every `call hist_addfld1d`, `hist_addfld2d`, or
   `hist_addfld_decomp` call in `src/` registers a field. The name is the
   first argument and the long name is in `long_name=`.

   ```bash
   grep -rn "call hist_addfld" src/ | head
   ```

2. **`bld/namelist_files/namelist_definition_ctsm.xml`** lists which fields
   are on by default for a given physics version inside the
   `<entry id="hist_fincl1" ...>` defaults block.

3. **Run a case for one timestep** with `hist_empty_htapes = .true.` and an
   open `hist_fincl1`, then `ncdump -h` the resulting `h0a` file to see what
   was written. (CIME's `check_input_data` does not list history fields.)

## Where each field lives

History fields are added near where the underlying state is updated. To
find the file:

```bash
grep -rn "hist_addfld.* '<FIELDNAME>'" src/
```

Examples:

| Field | File |
|-------|------|
| `GPP` | `src/biogeochem/CNVegCarbonFluxType.F90` |
| `TLAI`, `TSAI` | `src/biogeophys/CanopyStateType.F90` |
| `H2OSOI`, `SOILLIQ`, `SOILICE` | `src/biogeophys/WaterStateBulkType.F90` |
| `TSOI`, `TSA` | `src/biogeophys/TemperatureType.F90` |
| `BTRAN` | `src/biogeophys/SoilMoistStressMod.F90` |
| `EFLX_LH_TOT`, `FSH` | `src/biogeophys/EnergyFluxType.F90` |
| `QRUNOFF` | `src/biogeophys/Wateratm2lndType.F90` (or per-runoff component) |
